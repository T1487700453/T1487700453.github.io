一、Ubuntu中设置静态IP

1.Ubuntu版本18之前

1.1 查看虚拟机网关

1.2 使用管理员权限打开 /etc/network/interfaces文件

sudo vim /etc/network/interfaces

1.2.1 此处需要输入密码，密码为briup

1.3 在该文件中添加如下内容，并保存

auto eth0
iface eth0 inet static
address 192.168.142.111
netmask 255.255.255.0
gateway 192.168.142.2

1.4 设置DNS配置文件

sudo vim /etc/resolvconf/resolv.conf.d/base

1.4.1 在文件中添加如下内容，第一个IP要与查出来的网关一致

1.5 重启网络配置和网络服务

/etc/init.d/networking restart

1.6 重启Ubuntu系统

reboot

1.7 关机

1.7 启动后使用 ifconfig 命令查看IP是否固定成功

二、更改服务器名称

sudo vim /etc/hostname

三、更改服务器名字和IP的对应关系

sudo vim /etc/hosts

在Windows中填写服务器名称和ip的对应关系

C:\Windows\System32\drivers\etc\hosts

修改目录权限

sudo chown -R briup:briup /opt/*

四、设置集群间的免密登录

  1. 在A和B两台主机上都执行ssh-keygen命令，生成公钥id_rsa.pub和私钥id_rsa

  2. 将A主机上的公钥通过某种方式拷贝给B主机，在 A 主机执行：

    scp -r .ssh/id_rsa.pub briup@192.168.142.101:~

  3. 在B主机上，将A主机的公钥添加(追加)到认证密钥文件中 ，在 B 主机执行：

    cat ~/id_rsa.pub >> ~/.ssh/authorized_keys

  4. 配置主机名和IP地址的映射关系：

    1. 在每一台主机上执行以下命令：

      sudo vi /etc/hosts

    2. 在每台主机的hosts文件中添加以下内容：

      127.0.0.1        localhost
192.168.142.101    hadoop101
192.168.142.102    hadoop102
192.168.142.103    hadoop103

    3. 最好将上述配置在Windows中也配置一下：

      C:\Windows\System32\drivers\etc\hosts

  5. 在每台主机上创建存放软件和软链接目录：

    1. 创建目录：

      sudo mkdir -p /opt/softwares /opt/links

    2. 修改权限：

      sudo chown -R briup:briup /opt/*

五、配置 jdk

  1. 解压JDK的安装包到/opt/softwares目录中

    tar zxvf jdk-8u261-linux-x64.tar.gz -C /opt/softwares/

  2. 创建JDK安装路径的软链接到/opt/links目录中

    ln -sf /opt/softwares/jdk1.8.0_261 /opt/links/jdk

  3. 配置环境变量

    1. 编辑文件 ~/.bashrc

      vim  ~/.bashrc

    2. 添加以下内容

      ####__JAVA_CONF__####
export JAVA_HOME=/opt/links/jdk
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib
export PATH=${PATH}:${JAVA_HOME}/bin:${JRE_HOME}/bin

    3. 执行以下命令使之生效

      source ~/.bashrc

  4. 从A服务器上传文件到B服务器

    scp -r /opt/softwares/* briup@us2:/opt/softwares/

六、Hadoop 的安装和配置

1.伪分布式Hadoop集群

1.1 搭建

  1. 解压

    tar zxvf hadoop-2.10.0.tar.gz -C /opt/softwares/

  2. 创建软链接

    ln -sf /opt/softwares/hadoop-2.10.0/ /opt/links/hadoop

  3. 编辑文件 ~/.bashrc

    vim  ~/.bashrc

  4. 配置环境变量

    ####_HADOOP_CONF_####
export HADOOP_HOME=/opt/links/hadoop
export HADOOP_USER_NAME=briup
export PATH=${PATH}:${HADOOP_HOME}/bin:${HADOOP_HOME}/sbin

  5. 执行以下命令使之生效

    source ~/.bashrc

  6. 修改/opt/links/hadoop/etc/hadoop目录下的配置文件

    1. 修改hadoop-env.sh文件

      • 将文件中的JAVA_HOME路径更换为绝对路径

    2. 修改core-site.xml文件

      <?xml version="1.0" encoding="UTF-8"?>

<configuration>
        <property>
                <name>fs.defaultFS</name>
                <value>hdfs://hadoopPD:9000</value>
        </property>
        <!-- 新增加的配置 -->
        <property>
                <name>hadoop.proxyuser.briup.hosts</name>
                <value>*</value>
        </property>
        <property>
                <name>hadoop.proxyuser.briup.groups</name>
                <value>*</value>
        </property>
        <property>
                <name>hadoop.http.staticuser.user</name>
                <value>briup</value>
        </property>
</configuration>

    3. 修改hdfs-site.xml文件

      <?xml version="1.0" encoding="UTF-8"?>
    dfs.nameservices hadoop-cluster dfs.replication 3 dfs.blocksize 32M dfs.namenode.name.dir file:///home/briup/.bigdata/hadoop/data/nn dfs.datanode.data.dir file:///home/briup/.bigdata/hadoop/data/dn dfs.namenode.checkpoint.dir file:///home/briup/.bigdata/hadoop/data/snn dfs.namenode.checkpoint.edits.dir file:///home/briup/.bigdata/hadoop/data/snn dfs.web.ugi briup,briup fs.permissions.umask-mode 000 dfs.permissions.enabled false dfs.permissions.superusergroup briup dfs.namenode.safemode.threshold-pct 0f ```

    1. 修改mapred-site.xml文件

      <?xml version="1.0" encoding="UTF-8"?>
    mapreduce.framework.name yarn mapreduce.jobhistory.address hadoopPD:10020 mapreduce.jobhistory.webapp.address hadoopPD:19888 mapreduce.job.ubertask.enable true ```

    1. 修改yarn-site.xml文件

      <?xml version="1.0" encoding="UTF-8"?>
    yarn.resourcemanager.hostname hadoopPD yarn.nodemanager.aux-services mapreduce_shuffle yarn.nodemanager.local-dirs file:///home/briup/.bigdata/hadoop/data/nm yarn.log.server.url http://hadoopPD:19888/jobhistory/logs/ yarn.log-aggregation-enable true yarn.web-proxy.address hadoopPD:54321 yarn.log-aggregation.retain-seconds -1 yarn.nodemanager.remote-app-log-dir /logs yarn.nodemanager.pmem-check-enabled false yarn.nodemanager.vmem-check-enabled false

2.全分布Hadoop集群

有多个计算机节点，主节点和从节点的服务器启动在不同的计算机节点上

2.1 搭建

  1. 在 Hadoop 的集群中，每个节点的配置都一模一样（可以参照伪分布的配置）

  2. 在 Hadoop 安装目录下的 etc/hadoop 目录中的 slaves 文件中填写所有从节点的 IP 地址或者主机名

    hadoop102
hadoop103

  3. 将hadoop101 /opt/softwares/hadoop-2.10.0 拷贝到 hadoop102、hadoop103

    scp -r /opt/softwares/hadoop-2.10.0 briup@hadoop102:/opt/softwares/
scp -r /opt/softwares/hadoop-2.10.0 briup@hadoop103:/opt/softwares/

  4. 分别建立软链接，配置环境变量

2.2 启动

  1. 在使用之前（第一次启动前）先进行格式化，但是该操作只执行一次

    hdfs namenode -format

  2. 一键启动（只能在主节点执行）

    # HDFS集群
    start-dfs.sh
    stop-hdfs.sh

# YARN集群
    start-yarn.sh
    stop-yarn.sh

# HDFS和YARN集群
    start-all.sh
# mapreduce 运行历史日志服务器    
    mr-jobhistory-daemon.sh start historyserver

  3. 在 hadoop101 ，启动主节点

    # 启动/停止主节点
# HDFS集群
    hadoop-daemon.sh start/stop namenode
# YARN集群
    yarn-daemon.sh start/stop resourcemanager
# mapreduce 运行历史日志服务器    
    mr-jobhistory-daemon.sh start historyserver

  4. 在 hadoop102、hadoop103 ，启动从节点

    # 启动/停止从节点
# HDFS集群
    hadoop-daemon.sh start/stop datanode
# YARN集群
    yarn-daemon.sh start/stop nodemanager

2.3 WEB监控页面

# HDFS集群
    http://hadoop101:50070

七、Ubuntu20下安装Mysql数据库

1.安装默认版本

# 更新软件源
sudo apt-get update
# 安装mysql
sudo apt-get install mysql-server

2.确认安装是否成功

service mysql status

  • 出现图片中内容说明安装成功，退出命令Ctrl+C

3.查询当前使用的mysql版本

mysql --version
mysql -V

4.mysql基本使用

4.1 启动mysql服务器

# 使用mysql默认安装，安装完成就启动了mysql

# 手动启动
service mysql start

4.2 确认mysql是否启动成功

sudo netstat -tap | grep mysql
# 节点处于 LISTEN 状态表示启动成功

4.3 进入mysql shell界面

4.3.1 如果安装过程中设置了密码

-- 方式一
mysql -u root -p
-- 出现 Enter password: 后输入密码
briup

-- 方式二
mysql -u root -pbriup
-- 密码与-p之间不能有空格
--  -u 后面跟连接的数据库用户
--  -p 后面跟密码

-- 命令结束符 “;” 或者 “/g”

mysql -h 127.0.0.1 -P 3306 -u root -pbriup mytest
    -h 指定地址
    -P（大写）指定端口号
    -u 后面是用户名
    -p（小写）后面是密码（-p和密码之间不能空格，不然就会理解为密码第一位是空格）
    mytest 选择使用的数据库

4.3.2 如果安装过程中未提示设置密码

mysql -u root

4.3.3 密码查询及更改

sudo cat /etc/mysql/debian.cnf
mysql -u debian-sys-maint -p
# 会提示输入密码
# 此处需要输入的密码是上面红框中框中的密码
# 修改密码
# 8.04版本
alter user 'root'@'localhost' identified with mysql_native_password by 'root';

# 8.04之前版本
SET PASSWORD=PASSWORD(’[修改的密码]’)
# 或
update user set password=password(‘123456’) where user=‘root’;

4.3.4 设置任意IP访问数据库

mysql> use mysql; 
mysql> update user set host = '%' where user = 'root'; 
mysql> select host, user from user; 
mysql> flush privileges;
# 1.修改Ubuntu中MySQL的配置文件；
sudo vi /etc/mysql/mysql.conf.d/mysqld.cnf

        port                    = 8021
        character-set-server    = UTF8
        bind-address            = 0.0.0.0

sudo service mysql restart

# 2.修改Ubuntu中MySQL的用户表中存储的root的用户信息，让其允许远程登录；
mysql -u root -p

mysql> update mysql.user set host='%' where user='root';

mysql> flush privileges;

    nc localhost 44444

# Name the components on this agent

a1.sinks = k1
a1.channels = c1

十二、Hive的安装与配置

1. 安装

  1. 解压

    tar zxvf apache-hive-3.1.2-bin.tar.gz -C /opt/softwares/

mv apache-hive-3.1.2-bin/ hive-3.1.2

  2. 创建软链接

    ln -sf /opt/softwares/hive-3.1.2/ /opt/links/hive

  3. 编辑文件 ~/.bashrc

    vim  ~/.bashrc

  4. 配置环境变量

    ####__HIVE_CONF__####
export HIVE_HOME=/opt/links/hive
export PATH=${PATH}:${HIVE_HOME}/bin
alias beeline='beeline --color=true --hiveconf hive.server2.logging.operation.level=NONE'

  5. 执行以下命令使之生效

    source ~/.bashrc

  6. 在 /opt/links/hive/conf 文件夹下配置相关文件

    1. 批量替换文件名 

      sudo apt install rename

rename 's/\.template$//' *.template

    2. 编辑 hive-env.sh文件

      HADOOP_HOME=/opt/links/hadoop

export HIVE_CONF_DIR=/opt/links/hive/conf

    3. 编辑 hive-exec-log4j2.properties和 hive-log4j2.properties 文件

      property.hive.log.dir = /opt/links/hive/logs/${sys:user.name}

    4. 编辑 llap-daemon-log4j2.properties文件

      property.llap.daemon.log.dir = /opt/links/hive/logs/${sys:user.name}

    5. 编辑 llap-cli-log4j2.properties 文件

      property.hive.log.dir = /opt/links/hive/logs/${sys:user.name}

    6. 编辑 hive-site.xml 文件

      <?xml version="1.0" encoding="UTF-8"?>
<configuration>
<!-- 配置hive中的表在hdfs中的存储地址，默认地址 /user/hive/warehouse-->
<property>
<name>hive.metastore.warehouse.dir</name>
<value>/hive/warehouse</value>
</property>

<property>
<name>hive.metastore.db.type</name>
<value>mysql</value>
</property>
<property>
<name>javax.jdo.option.ConnectionURL</name>
<value>jdbc:mysql://us1:3306/hive?createDatabaseIfNotExist=true</value>
</property>
<property>
<name>javax.jdo.option.ConnectionDriverName</name>
<value>com.mysql.cj.jdbc.Driver</value>
</property>
<property>
<name>javax.jdo.option.ConnectionUserName</name>
<value>root</value>
</property>
<property>
<name>javax.jdo.option.ConnectionPassword</name>
<value>root</value>
</property>
</configuration>

    7. 将mysql依赖 添加到 opt/links/hive/lib 目录下

      mv mysql-connector-java-8.0.16.jar /opt/links/hive/lib

    8. 在 /opt/links/hive/bin 中 新建文件 start-hive.sh

      #!/bin/bash

nohup hive --service metastore > /dev/null 2>&1 &
nohup hive --service hiveserver2 > /dev/null 2>&1 &

    9. 给 start-hive.sh 赋予执行权限

      chmod +x start-hive.sh

    10. 在 /opt/links/hive/bin 中 新建文件 stop-hive.sh

      #!/bin/bash

ps -ef | grep RunJar | awk '{print $2}' | xargs kill -9 > /dev/null 2>&1

    11. 给 stop-hive.sh 赋予执行权限

      chmod +x stop-hive.sh

    12. MySQL中 初始化 元数据

      schematool -dbType mysql -initSchema

    13. 配置hive支持事务，在/opt/links/hive/bin 下的start-hive.sh文件，添加如下配置

      <!-- hive有关事务的配置  --> 
<property>
<name>hive.txn.manager</name>
<value>org.apache.hadoop.hive.ql.lockmgr.DbTxnManager</value>
</property>
<property>
<name>hive.compactor.initiator.on</name>
<value>true</value>
</property>
<property>
<name>hive.support.concurrency</name>
<value>true</value>
</property>
<property>
<name>hive.compactor.worker.threads</name>
<value>2</value>
</property>
<property>
<name>hive.exec.dynamic.partition.mode</name>
<value>nonstrict</value>
</property>
<!-- 设置在Web页面中查看某个操作执行过程  --> 
<property>
<name>hive.log.explain.output</name>
<value>true</value>
</property>

2. 启动

  1. 启动 元数据库 的进程

    hive --service metastore

# 后台运行
nohup hive --service metastore > /dev/null 2>&1 &        # hivemetastore

  2. 启动 服务器端

    hive --service hiveserver2

# 后台运行
nohup hive --service hiveserver2 > /dev/null 2>&1 &        #hiveserver2

  3. 使用客户端连接

    beeline -u jdbc:hive2://us1:10000

# 日志打印到后台
beeline -u jdbc:hive2://us1:10000 --color=true --hiveconf hive.server2.logging.operation.level=NONE
# 或
beeline --color=true --hiveconf hive.server2.logging.operation.level=NONE
beeline> !connect jdbc:hive2://us1:10000

# 配置beeline后
beeline -u jdbc:hive2://us1:10000 -n briup

    1. 安装 net-tools

      sudo apt install net-tools

    2. 监控端口

      netstat -anop

  4. 退出客户端

    !exit

3. 监控页面

us1:10002

1.3 简单使用

// 在 scala> 命令行使用
val file = sc.textFile("file:///etc/passwd")
val rdd1 = file.flatMap(_.split(":"))
val rdd2 = rdd1.map((_,1))
val res = rdd2.reduceByKey(_+_)

// 打印结果（行动操作）
res.foreach(println)    // 词频统计

// 整理为一行
sc.textFile("file:///etc/passwd").flatMap(_.split(":")).map((_,1)).reduceByKey(_+_).foreach(println)

val file = spark.read.textFile("file:///etc/passwd")
val res = file.flatMap(line => line.split(":")).groupByKey(identity).count()
res.show
